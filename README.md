# Модуль BL0937_Energy для чипа LN882HK (платформа LibreTiny)

Универсальный изолированный модуль неблокирующего энергомониторинга на чипе BL0937 (HLW8012) для умных розеток и реле [INDEX].

## Особенности модуля
* **Неблокирующая логика:** Автомат переключения таймингов работает на системном счетчике `millis()` без задержек `delay()`, что исключает конфликты с Blynk или WebOTA [INDEX].
* **Безопасность портов:** Прерывания привязаны к изолированным Си-функциям верхнего уровня, что защищает силовое реле на Порту B от аппаратного заклинивания [INDEX].
* **Гибкость настроек:** Калибровочные коэффициенты и интервалы опроса имеют встроенные дефолтные значения, но могут динамически перестраиваться из внешнего кода [INDEX].
* **Чистая архитектура:** Модуль отвечает строго за физические параметры (U, I, P), оставляя подсчет киловатт-часов (`total_kwh`) под полным контролем разработчика в главном файле [INDEX].

## Быстрый запуск в проекте

В файле `src / main.cpp` инициализируйте модуль и запустите автомат опроса:

```cpp
#include <Arduino.h>
#include <BL0937_Energy.h>

BL0937_Energy sensor; // Создание объекта измерителя

void setup() {
    // 1. Запуск датчика на физических пинах розетки (CF, CF1, SEL)
    sensor.begin(20, 21, 22);
    
    // 2. Опционально: установка кастомной калибровки. 
    // Если функцию не вызывать, автоматически применятся стабильные дефолтные значения.
    sensor.setCalibration(0.132, 0.01, 1.177);
}

void loop() {
    // 3. Поддержание работы автомата опроса чипа в фоне
    sensor.update(); 
    
    // 4. Получение живых данных в любой момент времени:
    float voltage = sensor.getVoltage(); // Напряжение (Вольты)
    float current = sensor.getCurrent(); // Ток (Амперы)
    float power   = sensor.getPower();   // Мощность (Ватты)
}
```

## 📖 Полный справочник функций и параметров (API)

### 1. Метод `begin()` — Инициализация железа
Вызывается строго в блоке `void setup()` один раз при старте контроллера [INDEX].
* **Синтаксис:** `sensor.begin(cfPin, cf1Pin, selPin);`
* **Входные параметры:**
  * `cfPin` (тип `uint8_t`) — Номер GPIO для импульсов мощности (лапа 25) [INDEX].
  * `cf1Pin` (тип `uint8_t`) — Номер GPIO для импульсов тока/напряжения (лапа 26) [INDEX].
  * `selPin` (тип `uint8_t`) — Номер GPIO управления режимом чипа (лапа 27) [INDEX].

### 2. Метод `update()` — Работа автомата опроса
Вызывается строго на каждом витке главного цикла `void loop()` [INDEX].
* **Синтаксис:** `sensor.update();`
* **Принцип работы:** Проверяет таймер `millis()`. Каждые 500 мс считывает импульсы, обновляет показатели и переключает ногу `SEL` (0 — Напряжение, 1 — Ток) [INDEX].

### 3. Метод `setCalibration()` — Настройка точности (Калибровка)
Позволяет передать индивидуальные коэффициенты под конкретный экземпляр розетки [INDEX]. 
* **Синтаксис:** `sensor.setCalibration(vCoeff, iCoeff, pCoeff);`
* **Параметры по умолчанию:**
  * `vCoeff` (тип `float`, по умолчанию `= 0.132f`) — Множитель для Вольт [INDEX].
  * `iCoeff` (тип `float`, по умолчанию `= 0.01f`) — Множитель для Ампер [INDEX].
  * `pCoeff` (тип `float`, по умолчанию `= 1.177f`) — Множитель для Ватт [INDEX].

### 4. Метод `setMeasureInterval()` — Скорость переключения режимов
Задает время удержания пина `SEL` для накопления импульсов тока или напряжения [INDEX].
* **Синтаксис:** `sensor.setMeasureInterval(intervalMs);`
* **Входные параметры:** `intervalMs` (тип `uint32_t`, по умолчанию `= 500`) [INDEX].

### 5. Функции получения данных (Геттеры)
* **`sensor.getVoltage()`** — Возвращает Напряжение. Тип: `float`. Величина: **Вольты (В)** [INDEX].
* **`sensor.getCurrent()`** — Возвращает Ток. Тип: `float`. Величина: **Амперы (А)** [INDEX].
* **`sensor.getPower()`** — Возвращает Активную мощность. Тип: `float`. Величина: **Ватты (Вт)** [INDEX].

## 🧭 Пошаговый порядок интеграции в проект (Для новичков)

1. **Объявление:** Подключите файл `#include <BL0937_Energy.h>` и создайте объект `sensor` на самом верху вашего кода, вне каких-либо функций [INDEX].
2. **Конфигурация:** Внутри `void setup()` настройте стандартную периферию розетки (`pinMode` кнопки, реле и светодиода) [INDEX].
3. **Запуск измерителя:** Строку `sensor.begin(20, 21, 22);` вызывайте в самом конце блока `setup()`, строго **ПОСЛЕ** успешного запуска Wi-Fi и Blynk, чтобы избежать аппаратных конфликтов регистров FreeRTOS [INDEX].
4. **Тактование:** Внутри `void loop()` первым делом ставьте `sensor.update();`, чтобы автомат опроса работал непрерывно [INDEX].
5. **Вывод данных:** Запрос геттеров (`sensor.getVoltage()` и др.) производите внутри ваших неблокирующих таймеров (например, раз в 2 секунды), чтобы не перегружать каналы связи Blynk или веб-сервера [INDEX].

## 🔌 Физика критериев измерения (Поведение железа)
* Микросхема BL0937 измеряет параметры только при наличии физического питания на силовой шине.
* Когда реле розетки закрыто, ток через измерительный шунт не течет, и чип перестает генерировать импульсы по линиям тока и мощности. В этот момент геттеры `getCurrent()` и `getPower()` будут выдавать чистый `0.0` [INDEX].
* Напряжение измеряется параллельно силовой линии, поэтому `getVoltage()` будет исправно показывать сетевые `220 В` даже при выключенной розетке [INDEX].
